Obesity

Data:

The data used is the obesity rate of each city in percentage value.
The data attribute type is Quantitative and ratio data, arithmetic can be performed to calculate and compare the difference in obesity level for the different cities. The data has a sequential ordering and is used to present the magnitude.

Task:

The visualisation aims to present obesity levels. It is mainly exploratory analysis and allows users to discover which are the cities that have either high/low obesity levels.

Idiom:

Using a Bubble chart visual encoding, we encoded the obesity levels for each city. We also leverage colour as a means to convey quantitative difference using the intensity. The obesity percentage for each city is also used as a size scaling factor. Cities with higher obesity will appear to have a bigger area to the viewer.

Obesity x Gym cost

Data:

The data used are obesity levels of each of each city in percentage value and cost of monthly gym membership in euro currency. The data attribute types are qualitative for both the data.

Task:

The visualisation aims to present the correlation between the obesity level and gym membership cost. Exploratory analysis approach is used to present the data.

Idiom:

Using a scatter plot, we encoded obesity level and gym cost for each city. We encode each city with colour to differentiate the cities. The obesity level and gym cost is also encoded as scales on the y and x axis.

Happiness Level

Data:

The data is happiness levels ranked based on score value. The data attribute type is quantitative

Task:

The visualisation aims to present the city's happiness level by ranking it.

Idiom:

Using a bar chart to display the relationship between the city happiness level and and city. The length of each bar represents the happiness level of each city

Happiness Level x Life Expectancy

Data:

The data used are both quantitative, and the data scale for happiness level and life expectancy are ratio and interval data, respectively.

Task:

The objective is to present and analyse correlation between the happiness level and life expectancy of the different cities.

Idiom:

Visual encoding used is a scatterplot with a regression line, which allows us to derive any patterns/correlation. In the graph, the regression line shows that there exists a strong relationship between the two different attributes.

Sunshine hours

Data:

The data is amount of sunshine in hours. The data attribute type is quantitative. The average amount of sunshine hours is calculated as the mean of all data points

Task:

The visualisation aims to show the rankings of cities by sunshine hours and to show which cities have less than average sunshine hours

Idiom:

Using a Radial Bar chart (which kind of looks like a sun), we encoded the amount of sunshine hours for each city. The length of each bar represents the total amount of sunshine hours - longer is more. The average amount of sunshine hours is a position on a common scale (although it is the only one on the scale)
